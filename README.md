# AqylMoney — Tech Vision 2026, трек FinTech & Literacy

Telegram-бот по финансовой грамотности для школьников и студентов Казахстана:

- 🛡 **Antifraud** — проверка сообщений/объявлений на признаки финансового мошенничества (флагманский модуль: rule-based weighted scoring + опциональное объяснение от Claude)
- 💰 **Бюджет** — простой трекер доходов/расходов
- 📈 **Инвест-симулятор** — риск-профиль → модельный портфель на реальных рыночных данных (yfinance), деньги виртуальные

## 1. Установка

```bash
python -m venv .venv
.venv\Scripts\activate      # Windows
pip install -r requirements.txt
```

## 2. Получить Telegram Bot Token

1. Откройте Telegram, найдите **@BotFather**
2. Отправьте `/newbot`, придумайте имя и username (должен заканчиваться на `bot`)
3. Скопируйте выданный токен

## 3. Настроить .env

```bash
copy .env.example .env
```

Откройте `.env` и вставьте:

```
TELEGRAM_BOT_TOKEN=<токен от BotFather>
ANTHROPIC_API_KEY=<опционально, ключ Claude API>
```

Без `ANTHROPIC_API_KEY` antifraud-модуль работает полностью автономно (rule-based объяснения), просто без сгенерированных на естественном языке пояснений.

## 4. Запуск

```bash
python run.py
```

Откройте бота в Telegram и отправьте `/start`.

## Архитектура (для TVEP / код-ревью)

```
bot/
├── config.py            — загрузка .env
├── db.py                 — SQLite (aiosqlite): users, transactions, portfolios, holdings, fraud_checks
├── states.py              — FSM-состояния диалогов
├── keyboards.py            — inline-клавиатуры
├── handlers/
│   ├── start.py            — /start, главное меню
│   ├── antifraud.py         — флагманский модуль
│   ├── budget.py            — бюджет
│   └── invest.py            — риск-квиз + портфель
└── services/
    ├── fraud_scoring.py      — weighted rule-engine (не LLM-обёртка!)
    ├── llm_client.py          — Claude API, генерирует объяснение поверх готовой оценки
    ├── market_data.py         — реальные цены через yfinance
    └── portfolio.py           — риск-квиз, модельные портфели
```

**Почему antifraud — не "пустая обёртка над API":** оценка риска считается независимым от LLM
взвешенным алгоритмом (`fraud_scoring.py`) по конкретным сигналам (нереалистичная доходность, давление
срочностью, запрос конфиденциальных данных, схемы "лёгких денег", подозрительные ссылки). Claude
используется только для генерации понятного объяснения уже готового результата — при отсутствии ключа
бот использует шаблонное объяснение и работает идентично по логике.
