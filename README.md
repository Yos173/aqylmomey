# AqylMoney — Tech Vision 2026, трек FinTech & Literacy

Telegram-бот по финансовой грамотности для школьников и студентов Казахстана. Работает и как обычный
чат-бот на инлайн-кнопках, и как **Telegram Mini App** (полноценный веб-интерфейс внутри Telegram).

- 🛡 **Antifraud** — проверка сообщений/объявлений/скриншотов на признаки финансового мошенничества
  (флагманский модуль: rule-based weighted scoring + опциональная независимая оценка от Claude —
  итоговый риск берётся как максимум из двух)
- 💰 **Бюджет** — простой трекер доходов/расходов
- 📈 **Инвест-симулятор** — риск-квиз → модельный портфель, раздел «Рынки» с реальными котировками
  (акции, ETF, казахстанские бумаги) с автообновлением цен, ручная покупка/продажа на виртуальные деньги
- 🏆 **Геймификация** — финансовый IQ-квиз с разбором ошибок, бэйджи, публичный рейтинг по школам
- 🔴 **Fraud Radar** — открытая агрегированная статистика по самым частым схемам мошенничества
- 🤖 **AI-помощник** — коуч по финансам, советник по мошенничеству и обозреватель рынков в одном чате

## 1. Установка

```bash
python -m venv .venv
.venv\Scripts\activate      # Windows
pip install -r requirements.txt
```

## 2. Получить Telegram Bot Token

1. Откройте Telegram, найдите **@BotFather**
2. Отправьте `/newbot`, придумайте имя и username (должен заканчиваться на `bot`)
3. Скопируйте выданный токен

## 3. Настроить .env

```bash
copy .env.example .env
```

Откройте `.env` и вставьте:

```
TELEGRAM_BOT_TOKEN=<токен от BotFather>
ANTHROPIC_API_KEY=<опционально, ключ Claude API>
```

Без `ANTHROPIC_API_KEY` antifraud-модуль работает полностью автономно (rule-based объяснения), просто без сгенерированных на естественном языке пояснений.

## 4. Запуск

```bash
python run.py
```

Откройте бота в Telegram и отправьте `/start`. Без `WEBAPP_URL` в `.env` бот работает как обычный
чат-бот на инлайн-кнопках — Mini App не требуется, чтобы попробовать функциональность.

## 5. (Опционально) Открыть бота как Telegram Mini App

Telegram принимает только HTTPS-адреса для Mini App, `localhost` не подходит — для локальной разработки
нужен туннель, например [ngrok](https://ngrok.com/download):

```bash
ngrok http 8080
```

Ngrok выдаст адрес вида `https://xxxx.ngrok-free.app`. Впишите его в `.env`:

```
WEBAPP_URL=https://xxxx.ngrok-free.app
```

Перезапустите `python run.py` — теперь `run.py` поднимает и polling-бота, и веб-сервер (FastAPI/uvicorn,
порт из `WEBAPP_PORT`) в одном процессе, а `/start` присылает кнопку «🚀 Открыть AqylMoney», которая
открывает веб-интерфейс прямо внутри Telegram. Также настраивается постоянная кнопка меню в чате.

**Важно:** при каждом перезапуске ngrok (бесплатный тариф) адрес меняется — придётся обновлять
`WEBAPP_URL` и перезапускать бота.

## 6. Постоянный деплой (Render.com)

Локальный запуск + туннель (ngrok/localtunnel) подходят для разработки и живого демо с вашего компьютера,
но ссылка живёт только пока запущен ваш процесс. Чтобы жюри могли открыть сайт **в любое время, без вас**,
нужен настоящий хостинг. В репозитории уже есть `render.yaml` — конфиг для бесплатного тарифа Render.

1. Залейте код на GitHub (если ещё не сделано):
   ```bash
   git remote add origin https://github.com/<ваш-логин>/<репозиторий>.git
   git push -u origin main
   ```
2. На [render.com](https://render.com) → **New** → **Blueprint** → укажите URL вашего GitHub-репозитория.
3. Render найдёт `render.yaml` и предложит ввести значения для секретных переменных
   (`TELEGRAM_BOT_TOKEN`, `ANTHROPIC_API_KEY`) — вставьте те же, что в вашем `.env`.
4. Нажмите **Apply** — через пару минут сервис будет доступен по адресу вида
   `https://aqylmoney.onrender.com` (или похожему, если имя занято).

**Важно про бесплатный тариф:**
- Сервис «засыпает» после ~15 минут без запросов — первый заход после простоя может занять 30-60 секунд
  (сервер просыпается), дальше работает быстро.
- Диск эфемерный — без Turso (см. ниже) локальный файл `aqylmoney.db` сбрасывается при каждом новом
  деплое (но не при засыпании/пробуждении).

**Чтобы данные (лидерборд, бэйджи, портфели) переживали деплои — Turso:**

Render не даёт подключить постоянный диск на бесплатном тарифе, поэтому база по умолчанию живёт в
локальном файле, который стирается при каждом деплое. Чтобы данные сохранялись насовсем, бот умеет
подключаться к бесплатной облачной базе [Turso](https://turso.tech) (libSQL, полностью совместим с SQLite):

1. Зарегистрируйтесь на [turso.tech](https://turso.tech), создайте базу (`turso db create aqylmoney`
   через их CLI, либо через веб-интерфейс).
2. Скопируйте **Database URL** (вида `libsql://aqylmoney-<org>.turso.io`) и создайте **auth token**.
3. В переменных окружения Render впишите `TURSO_DATABASE_URL` и `TURSO_AUTH_TOKEN` этими значениями и
   перезапустите сервис.

Без этих двух переменных всё продолжает работать как раньше — на локальном файле `DB_PATH`, это
полностью опционально и не требуется для локальной разработки.
- Порт сервер слушает автоматически через переменную `$PORT`, которую выставляет Render — `WEBAPP_PORT`
  из `.env` для этого не нужен, `bot/config.py` сам отдаёт приоритет `$PORT`, если она задана.
- Чтобы включить кнопку Telegram Mini App и на проде, впишите в переменные окружения Render
  `WEBAPP_URL=https://<ваш-сервис>.onrender.com` после первого деплоя и перезапустите сервис.

## Архитектура (для TVEP / код-ревью)

```
bot/
├── config.py            — загрузка .env / переменных окружения
├── db.py                 — libsql-client: локальный файл SQLite или облачная Turso (см. раздел 6);
│                            users, transactions, portfolios, holdings, fraud_checks, web_users,
│                            financial_iq_scores
├── states.py              — FSM-состояния диалогов
├── keyboards.py            — inline-клавиатуры
├── handlers/
│   ├── start.py            — /start, главное меню (или кнопка Mini App, если задан WEBAPP_URL)
│   ├── antifraud.py         — флагманский модуль
│   ├── budget.py            — бюджет
│   └── invest.py            — риск-квиз, рынки, покупка/продажа, портфель
├── services/
│   ├── fraud_scoring.py      — weighted rule-engine (не LLM-обёртка!)
│   ├── llm_client.py          — Claude API: гибридная оценка риска, объяснения, OCR скриншотов, AI-помощник
│   ├── market_data.py         — реальные цены через yfinance (акции, ETF, KZ-бумаги), с кэшем
│   ├── portfolio.py           — риск-квиз, модельные портфели
│   ├── financial_quiz.py      — вопросы IQ-квиза, скоринг, разбор ошибок
│   └── badges.py              — условия выдачи бэйджей
└── webapp/                  — Telegram Mini App + открытый сайт (опционально, см. раздел 5)
    ├── auth.py                — валидация подписи Telegram initData (HMAC) + веб-сессии по нику
    ├── server.py               — FastAPI: те же операции, что и в handlers/, плюс геймификация,
    │                              Fraud Radar, AI-помощник — как JSON API
    └── static/                 — фронтенд (vanilla HTML/CSS/JS, без сборки): landing.html (сайт),
                                   radar.html/radar.js (Fraud Radar), index.html/app.js (Mini App)
```

`bot/webapp/server.py` переиспользует `bot/db.py` и `bot/services/*` напрямую — бизнес-логика не
дублируется между чат-версией и Mini App, отличается только слой доставки (aiogram-хендлеры vs HTTP-роуты).

**Почему antifraud — не "пустая обёртка над API":** базовая оценка риска считается независимым от LLM
взвешенным алгоритмом (`fraud_scoring.py`) по конкретным сигналам (нереалистичная доходность, давление
срочностью, запрос конфиденциальных данных, схемы "лёгких денег", подозрительные ссылки) — и работает
полностью автономно без ключа Claude. При наличии ключа Claude даёт вторую, независимую оценку риска по
смыслу сообщения (а не по словам), и итоговый риск — это максимум из двух оценок: так безопаснее для
антифрода, если хотя бы один из "судей" считает сообщение опасным. Без ключа бот использует шаблонное
объяснение и работает чисто на правилах.
